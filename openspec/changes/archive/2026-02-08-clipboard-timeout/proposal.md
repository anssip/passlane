## Why

Passwords copied to the clipboard by the `show` command remain there indefinitely, creating a security risk — any application can read the clipboard, and users may accidentally paste credentials into unintended contexts. Auto-clearing the clipboard after 10 seconds limits this exposure window.

## What Changes

- After the `show` command copies a password to the clipboard, a background timer starts that clears the clipboard after 10 seconds.
- The user sees a message indicating the clipboard will be auto-cleared (e.g., "Password copied to clipboard! Clipboard will be cleared in 10 seconds.").
- Only the clipboard content set by Passlane is cleared — if the user has already copied something else before the timer fires, the clipboard is left untouched.
- Applies to credential passwords copied via `show` (single-match auto-copy and multi-match row selection) and to passwords generated by the `generate` command. Does not apply to payment card numbers, TOTP codes, or the `add` command.
- A new `--out` flag is added to `show` and `generate` commands. When set, the password is printed to STDOUT only — clipboard is not used at all, and no timeout applies. This enables scripting use cases (e.g., piping password output to other tools).

## Capabilities

### New Capabilities
- `clipboard-timeout`: Timed clipboard clearing after password copy in the `show` and `generate` commands, plus `--out` flag to skip clipboard entirely for scripting

## Impact

- **Code**: `src/actions/mod.rs` — the `copy_to_clipboard` function (or a new variant) needs to spawn a background thread/timer that clears the clipboard after 10 seconds.
- **Code**: `src/actions/show.rs` — `ShowCredentialsTemplate` methods that call `copy_to_clipboard` need to use the timed variant and update the user-facing message.
- **Code**: `src/actions/generate.rs` — the `generate` action's `copy_to_clipboard` call needs to use the timed variant and update the user-facing message.
- **Code**: `src/main.rs` — add `--out` flag to the `show` and `generate` subcommand definitions.
- **Dependencies**: No new crates expected — `std::thread` + `std::time::Duration` plus the existing `clipboard` crate suffice.
- **Platforms**: Behaviour should work on macOS, Linux, and Windows (all supported by the `clipboard` crate).
